4. Requirements for Every Change

Before implementation

  • Described and approved (Change Request – RFC).
  • Risk assessment – especially for key systems.
  • Test plan – how to verify correct operation after the change.
  • Rollback plan – how to revert the change if something goes wrong.
  • Backup before implementing changes to critical systems (DOC-012).

After implementation

  • Verification per test plan.
  • Update of configuration documentation.
  • Close RFC with result (success / partial success / rollback).
  • For emergency changes: full retrospective documentation within ▸ [24h].
